Project 2

Redesign

Assignment

Take a poorly designed website, analyze it and identify its flaws. Using this analyses, redesign the website by first creating low and high fidelity prototypes, followed by an actual webpage that addresses the problems of the original site.

Goals

The goals of this project include: refining HTML/CSS Skills, analyzing interfaces on the bases of accessibility , creating low /high fidelity mockups, and practice building responsive websites.

accessibility test findings

The WAIVE accessibility results for this website weren't terrible but they can definitely be improved. There were multiple warnings of small text, long alternative text, and suspisious link text; but overall there were no major errors. The website also did well in terms of ARIA, with 165 ARIA elemnts being highlighed by the WAIVE application.

Low-Fidelity Mockups

Laptop

The wireframe of the redesigned website improves upon the original in numerous ways. For one it takes all of the navigation menus, login forms and navbars from the left of the interface and centers them/ distributes them evenly accross the screen so that the interface is no longer poorly aligned.

The wireframe also uses a grid layout for all announcments/ posts so that the user has a clearer mapping of elements on the page.

Tablet

The tablet version of the low fidelity mockup functions very similarly to the desktop version. The main differences are, the navbar list is replaced with a bar icon that opens a dropdown menu to reduce clutter, and the spacing of elements on the page is slightly more compresed.

Mobile

The mobile version is similar to the tablet, the only difference being that there is no space for multiple columns, resulting in a singular column of elemnts as you scroll.

High-Fidelity Mockups

Laptop

The high-fidelity mockup both improves upon the original website and expands on the low-fidelity wire frame by adding details not captured in the wireframe. The high fidelity shows a unified color scheme that simplifies the interface and decreses the busyness of the original.

Each element laid out horizontally is in a flexbox with row wrap so that when the screen size changes, they will condense before wrapping into one single column.

Tablet

The tablet version is identical to the desktop with the only exception being the navbar. In the desktop all the navigation links are displayed but when the screen width decreases to tablet size, bootstrap is used to make the bar icon display and lead to a drop-down menu on-press.

Mobile

The mobile version is very similar to the tablet except that the width of the screen is now the smallest, which compressed the flex rows and results in all if the elements of the page being stored in a single column.

Visual Design Style Guide

The visual design guide is relatively simple; few colors are used in the interface.

Links are grey regularly, orange on hover, and black when selected. There is one icon used, and it is the navbar bar icon, used when the screen width is compressed. There is one style of button, which remains the same on click and on standby. There are two types of headers for posts, the main with the orange shadow and the secondary header with no shadow and same fill as main.
